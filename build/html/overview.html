Overview¶

This overview will focus on the implementation of Seldonian algorithms (SAs) in this library, rather than a description of what they are. For a detailed description of SAs see the UMass AI Safety page, specifically the Science paper.

Often in a normal ML problem, data are split into training, testing (and sometimes validation) sets. SAs split data into candidate selection data and safety test data. SAs consist of three main parts: the interface, candidate selection, and the safety test, which are described below.

Interface¶

In the interface, the user provides the data, metadata, the constraints that it wants the SA to enforce, and the confidence thresholds for each constraint. The interface creates a DataSet object containing the data and metadata. The interface also creates ParseTree objects from the interpreted constraints. Finally, the interface generates a Spec object which consists of a complete specification that can used to run the seldonian algorithm in its entirety. The Spec object that the interface creates has many default parameters that the user may modify with their own custom script.
